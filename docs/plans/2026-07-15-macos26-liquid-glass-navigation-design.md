# macOS 26 Liquid Glass、悬浮窗与导航优化设计

日期：2026-07-15

## 目标

在保留 macOS 15–25 兼容性的前提下，为 macOS 26 提供原生 Liquid Glass 视觉；点击停止录音后及时关闭悬浮窗；在会议详情页增加返回首页入口并始终保留历史会议侧栏；统一全局动画并尊重系统辅助功能设置。

## 已确认决策

- macOS 26 使用原生 `glassEffect`、`GlassEffectContainer`、`.glass` 与 `.glassProminent`。
- macOS 15–25 使用 `Material`、圆角描边和轻量阴影回退。
- 采用克制的系统原生风格：玻璃用于导航、工具栏、悬浮录音条、主要按钮与关键卡片，长篇转录正文不玻璃化。
- 详情页左上角提供“返回首页”，点击后清除当前会议选择；历史会议侧栏始终保留。
- 悬浮录音条仍严格只有录音、暂停/继续、停止、书签四个控件，不增加关闭键。

## 架构与组件

新增集中式自适应视觉层，统一管理：

- 自适应玻璃表面：macOS 26 使用原生 Glass，旧系统使用 Material 回退。
- 普通与强调按钮：macOS 26 使用原生玻璃按钮样式，旧系统使用现有 bordered 样式。
- 动画规范：统一页面切换、控件反馈和悬浮窗显隐的时长与曲线。

`RootView` 继续拥有会议选择状态，并向 `MeetingDetailView` 注入返回闭包。详情视图不直接持有会议列表或修改历史数据。

`FloatingPanelController` 负责面板淡入、淡出和最终 `orderOut`。`MeetingCoordinator` 在停止请求通过状态机校验、资源校验并成功将会议写为“处理中”后立即隐藏面板，随后继续完成音频停止、转录、落盘和会议终结。

## 交互与状态流

### 开始、暂停与书签

- 开始录音后，悬浮窗以短促淡入和轻微缩放出现。
- 暂停/继续只平滑更新对应按钮的图标、颜色和玻璃高光，不重建整个面板。
- 添加书签提供轻量按钮反馈，不改变面板尺寸。

### 停止录音

1. 状态机确认当前状态允许停止。
2. 验证活动会议、采集器、writer、转录器和时间线均存在。
3. 将会议持久化状态写为“处理中”，同步更新内存状态机。
4. 立即淡出并关闭悬浮窗。
5. 后台停止采集并完成 writer、转录队列和会议落盘。
6. 成功时进入可总结状态；失败时主窗口显示错误，悬浮窗保持关闭。

现有生命周期锁继续拦截重复停止请求。

### 返回首页

- 详情页左上角工具栏显示“返回首页”。
- 点击只把 `selectedMeetingID` 设为空，详情区回到“线下会议 / 在线会议”入口。
- 侧栏、会议列表、录音文件和选择之外的状态不变。
- 再次选择任意历史会议时，详情区平滑切换，侧栏不重建。

## 视觉与动画

- Liquid Glass 仅用于具有层级或操作意义的表面，避免所有内容区域同时模糊。
- 录音和停止使用红色语义 tint；暂停为橙色、继续为绿色、书签为蓝色。
- 页面选择切换采用约 0.2–0.3 秒的系统弹性动画。
- 悬浮窗关闭采用更短淡出，明确表达录音已停止。
- 动画只改变 `opacity`、`scale` 和系统 Glass transition，不持续驱动布局尺寸。
- 不增加计时器装饰动画，避免长录音期间产生额外能耗。

读取 SwiftUI 的 `accessibilityReduceMotion`：开启时取消缩放和弹性，只保留快速淡入淡出。降低透明度由原生 Glass 与 Material 自动适配。

## 错误处理

- 若停止请求在进入“处理中”之前失败，保持悬浮窗可见，用户仍可重试。
- 一旦成功进入“处理中”，悬浮窗立即关闭；其后的转录或落盘失败通过主窗口错误信息反馈，不重新弹出面板。
- 重复显隐必须重置面板透明度和变换，避免下次录音出现透明或不可点击的窗口。
- 返回首页不影响正在执行的总结、归档任务或已保存会议内容。

## 测试与验收

1. Coordinator 单元测试验证进入“处理中”后立即隐藏面板，并覆盖后续 writer、转录或落盘失败时面板仍关闭。
2. 悬浮窗测试继续保证严格四个控件，并验证重复显示/隐藏可恢复透明度与交互状态。
3. 导航测试验证返回首页只清除选择，历史会议保持不变。
4. 视觉策略测试验证 macOS 26 选择原生 Glass，macOS 15–25 选择 Material 回退。
5. UI 测试验证开始录音显示悬浮窗、点击停止后悬浮窗消失、返回首页后侧栏历史记录仍存在。
6. 运行完整单元测试、签名 UI 测试、arm64 构建，并在 macOS 26 真机检查玻璃效果、焦点环、VoiceOver、减少动态效果和主观流畅度。

自动化视觉测试只证明结构与状态正确；最终流畅度和层级效果仍需真实界面观感确认。
