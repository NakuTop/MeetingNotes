# 全窗口 Liquid Glass 最小补丁设计

## 目标

在不改动信息架构和业务逻辑的前提下，让主窗口、历史会议侧栏和主内容共享同一层 macOS 26 原生 Liquid Glass。macOS 15–25 继续使用系统 Material 回退。

## 实现边界

- 在 `RootView` 中用单一背景层覆盖整个主窗口，现有 `NavigationSplitView` 仍负责导航和侧栏语义。
- 背景层由 `AppVisualStyle` 提供：macOS 26 使用 `glassEffect(.regular)`，旧系统使用 `ultraThinMaterial`。
- 隐藏历史列表的不透明滚动背景，使共享玻璃层在侧栏中可见。
- 现有主按钮、返回按钮、录音浮窗和关键卡片保留为上层玻璃。长转录、书签和长总结区仍保持清晰的普通内容表面。
- 不更改历史会议数据、录音、转录、DeepSeek 或 Notion 流程。

## 交互与可访问性

共享背景不参与点击和键盘焦点，不改变现有辅助功能树。侧栏的选择、返回首页和四键浮窗保持原有语义。背景不新增动画，继续遵循现有 Reduce Motion 策略。

## 验证

- 先扩展 UI 测试，要求主窗口存在稳定的共享玻璃表面标识，并观察其在实现前失败。
- 实现后运行聚焦 UI 测试，再运行完整单元与 UI 回归。
- 在 macOS 26 Apple Silicon 上保留首页与详情截图，目视确认侧栏和主内容共享背景。
- 重新构建 macOS 15 部署目标的 arm64 App，并严格校验签名。
